UI Architecture Enhancements Sprint v1.0

Executive Summary

This sprint focuses on modernizing and optimizing the UI architecture by upgrading to Tailwind CSS v4, implementing Vite-native CSS processing, and establishing comprehensive design systems. These enhancements will improve build performance, developer experience, and design consistency across the application.

Sprint Objectives

1. Tailwind CSS v4 Migration & Optimization

Priority: High | Estimated Effort: 2-3 days

Current State

  • Project upgraded to Tailwind CSS v4.1.17
  • New dependencies installed: @tailwindcss/cli, @tailwindcss/postcss, @tailwindcss/vite
  • Using separate CLI build process alongside Vite
  • CSS files still using legacy Tailwind v3 syntax (@tailwind directives)

Objectives

  • Update app/styles/app.css to use Tailwind v4 CSS-first syntax
  • Migrate from @tailwind base/components/utilities to @import "tailwindcss"
  • Review and update tailwind.config.ts for v4 compatibility
  • Consolidate CSS variable definitions with v4's CSS-based configuration
  • Test all Tailwind utility classes and custom styles for compatibility

Implementation Details

// Current app/styles/app.css (v3 syntax) @tailwind base; @tailwind components; @tailwind utilities; @layer base { :root { --background: 222.2 84% 4.9%; // ... other variables } } // Target app/styles/app.css (v4 syntax) @import "tailwindcss"; @layer base { :root { --background: 222.2 84% 4.9%; // ... other variables } }

Success Criteria

  • All pages render correctly with v4 syntax
  • CSS bundle size remains optimized
  • No console errors or warnings related to Tailwind
  • Build process completes without deprecation warnings

2. Vite-Native CSS Processing Integration

Priority: High | Estimated Effort: 1-2 days

Current State

  • Using separate @tailwindcss/cli for CSS compilation
  • Build script: npx @tailwindcss/cli -i ./app/styles/app.css -o ./public/styles/tailwind.build.css
  • @tailwindcss/vite plugin installed but not actively used
  • Vite config has PostCSS setup with tailwindcss() and autoprefixer

Objectives

  • Migrate from CLI-based builds to Vite-native CSS processing
  • Configure @tailwindcss/vite plugin for automatic CSS compilation
  • Eliminate redundant build steps and improve hot module replacement (HMR)
  • Streamline development workflow with single build process

Implementation Plan

// vite.config.ts - Enhanced Configuration import { defineConfig } from "vite"; import { reactRouter } from "@react-router/dev/vite"; import tailwindcss from "@tailwindcss/vite"; import autoprefixer from "autoprefixer"; export default defineConfig({ plugins: [ reactRouter(), tailwindcss(), // ‚Üê Use Vite plugin for Tailwind ], css: { postcss: { plugins: [autoprefixer], }, }, // ...other config });
// package.json - Simplified Scripts { "scripts": { "dev": "concurrently \"react-router dev\"", "build": "react-router build", "build:css": "vite build --mode production", // Remove: "watch:css" - Handled by Vite HMR } }

Benefits

  • Faster development builds with integrated HMR
  • Single build process reduces complexity
  • Better source maps and debugging experience
  • Automatic CSS optimization in production builds

Success Criteria

  • CSS changes reflect instantly in dev mode without manual rebuild
  • Production build generates optimized CSS bundle
  • No need for separate watch:css or build:css commands
  • Build time improves by at least 20%
